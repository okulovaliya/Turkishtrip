# Turkey Challenge — настройка общих данных (Firebase)

По умолчанию приложение хранит всё в `localStorage` браузера — каждая подруга видит только свою активность на своём телефоне. Чтобы все трое видели активность друг друга (общая лента, командный прогресс), нужно подключить бесплатную базу Firebase Realtime Database. Настройка разовая, займёт 5–10 минут, делает её один человек.

## Шаг 1. Создать проект Firebase

1. Откройте [console.firebase.google.com](https://console.firebase.google.com) и войдите с любым Google-аккаунтом.
2. Нажмите **Add project** / «Создать проект», придумайте название (например, `turkey-challenge`), пройдите мастер (Google Analytics можно отключить — не нужен).

## Шаг 2. Включить Realtime Database

1. В левом меню откройте **Build → Realtime Database**.
2. Нажмите **Create Database**.
3. Выберите любой регион (например, ближайший к вам).
4. На шаге про правила безопасности выберите **Start in test mode** — это временно открывает чтение/запись всем, у кого есть ссылка на базу. Для узкого круга друзей этого достаточно; более строгие правила — см. раздел «Безопасность» ниже.

## Шаг 3. Получить конфигурацию веб-приложения

1. Нажмите на шестерёнку рядом с «Project Overview» → **Project settings**.
2. На вкладке **General** прокрутите до **Your apps** → нажмите значок `</>` (Web).
3. Введите любое имя приложения → **Register app**. Ничего дополнительно устанавливать не нужно.
4. Firebase покажет объект вида:

```js
const firebaseConfig = {
  apiKey: "AIza...",
  authDomain: "turkey-challenge-xxxxx.firebaseapp.com",
  databaseURL: "https://turkey-challenge-xxxxx-default-rtdb.firebaseio.com",
  projectId: "turkey-challenge-xxxxx",
  storageBucket: "turkey-challenge-xxxxx.appspot.com",
  messagingSenderId: "...",
  appId: "..."
};
```

5. Скопируйте эти значения в файл **`firebase-config.js`** (лежит рядом с `index.html`), заменив placeholder-строки. Нужен именно объект `firebaseConfig` — если Firebase выдал код с `import { initializeApp } from "firebase/app"`, эту строку и вызов `initializeApp(...)` не копируйте, оставьте только сам объект (`app.js` подключает Firebase самостоятельно).

## Шаг 4. Включить вход по паролю (Firebase Authentication)

Это то, что убирает пароли из файлов проекта — вместо этого их безопасно хранит и проверяет сам Firebase.

1. В левом меню откройте **Build → Authentication → Get started**.
2. На вкладке **Sign-in method** выберите **Email/Password** → включите переключатель **Enable** → **Save**.
3. Больше ничего создавать вручную не нужно: при первом входе каждая подруга сама придумывает себе пароль (от 6 символов) прямо в приложении — он автоматически регистрируется в Firebase. При следующих входах используется тот же пароль.

## Шаг 5. Разослать приложение подругам

Сохраните и раздайте всем троим одну и ту же папку (`index.html`, `style.css`, `app.js`, `users.json`, `firebase-config.js`) — например, залив на любой бесплатный статический хостинг (GitHub Pages, Netlify, Vercel) и разослав ссылку, либо просто переслав папку файлов, если открываете локально в Firefox (см. примечание про CORS ниже).

После этого на экране «Профиль» появится плашка **☁️ синхронизировано со всеми** — значит облако подключено. Если конфиг не заполнен, будет **📱 только на этом устройстве**.

## Как это работает

- `users.json` хранит только логин, имя, цвет и аватар каждой подруги — **паролей там больше нет**. Пароли целиком под управлением Firebase Authentication: он их хеширует и хранит на своей стороне, ни приложение, ни файлы проекта их никогда не видят.
- При первом входе под именем (например, `tanya`) любой введённый пароль от 6 символов автоматически регистрируется как её пароль. При всех следующих входах Firebase сверяет пароль сам и либо пускает, либо возвращает «Неверный пароль».
- В Firebase Realtime Database синхронизируются только сами данные челленджа: активность (шаги/тренировки), комментарии и реакции в ленте. Обновления приходят почти мгновенно на все устройства.
- Если `firebase-config.js` не настроен (стоят placeholder-значения), приложение переключается в локальный демо-режим на одном устройстве — без пароля вообще (см. предупреждение на экране входа). Как только конфиг заполнен и Email/Password включён в консоли — включается полноценный защищённый вход.

## Безопасность базы данных

По умолчанию (Realtime Database → Rules) Firebase создаёт **test mode** правила, которые открывают чтение и запись всем, у кого есть ссылка на базу, но только временно (обычно 30 дней от создания базы). После того как вы включили Firebase Authentication (Шаг 4), рекомендуется заменить их на правила, требующие входа:

```json
{
  "rules": {
    ".read": "auth != null",
    ".write": "auth != null"
  }
}
```

Так базу смогут читать и писать только те, кто реально вошёл в приложение под своим логином и паролем — а не кто угодно, кто узнал ссылку на базу. Обновите правила в Firebase Console → Realtime Database → вкладка **Rules** → вставьте JSON выше → **Publish**.

## Открытие без сервера

При открытии `index.html` напрямую двойным кликом (`file://`) браузер может блокировать `fetch('users.json')` из-за CORS — в этом случае сработает встроенный резервный список участниц в `app.js` (имена и цвета, без паролей), сам вход по-прежнему пойдёт через Firebase Authentication. Firebase-скрипты грузятся с CDN и не подвержены этому ограничению. Рекомендуется всё же открывать через любой локальный сервер (`python3 -m http.server`) или хостинг — так работает стабильнее.
